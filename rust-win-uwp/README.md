# foo-bar

A Rust sample that hosts a XAML window using the [`windows`](https://crates.io/crates/windows) and
[`windows-bindgen`](https://crates.io/crates/windows-bindgen) crates.

The window shows "Hello" centered above a `TextBox` + `Send` button. Press **Enter**
in the textbox or click **Send** to display a `MessageDialog` saying
`You said '<your text>'`. Dismissing the dialog clears the textbox and returns focus.
Click-and-drag anywhere in the window to move it around the screen.


## Layout

```bash
.
├── Cargo.toml                # crate manifest, windows-* feature flags
├── build.rs                  # calls windows_bindgen to emit src/bindings.rs
├── register.cmd              # build + register the MSIX package
├── appx/
│   ├── AppxManifest.xml      # package identity + visual elements
│   └── *.png                 # tile / store / splash artwork (placeholder)
└── src/
    ├── main.rs               # app entry point, CoreApplication bootstrap
    ├── bindings.rs           # generated by build.rs (do not edit)
    └── views/
        ├── main_view.rs      # XAML UI tree construction
        └── main_actions.rs   # event handlers (Send, Enter, drag)
```


## UWP Framework

This sample targets the UWP `Windows.UI.Xaml` stack.  The official `windows-rs` [`xaml_app`](https://FOO_GIT_REPO_DOMAIN/microsoft/windows-rs/tree/master/crates/samples/windows/xaml_app) sample (which this project extends) uses UWP XAML. 

## Prerequisites

- Rust (stable, MSVC toolchain)
  - x64: `rustup default stable-x86_64-pc-windows-msvc`
  - ARM64: `rustup default stable-aarch64-pc-windows-msvc`
- Visual Studio 2022 with the **Desktop development with C++** workload (needed for `link.exe` and the native tools command prompt for your architecture)
- Windows 10 1809+ / Windows 11
- Developer Mode enabled (Settings → For developers) so unsigned MSIX packages can be sideloaded with `Add-AppxPackage -Register`

## Build & run

UWP XAML apps cannot run as loose `.exe` files — they must be MSIX-registered first. 

Open a **Developer Command Prompt for VS 2022** (or the **ARM64** variant on ARM hardware) and run the below snippet.

```
cd C:\repo\path\foo-bar
register.cmd
```

To run it, tap the windows key and type:

```
Foo Bar Desktop
```

This should open the app from the start menu.  

To uninstall:

```
powershell -command "Get-AppxPackage *a3f8c2e1-9b4d-4e7a-b1c2-7d6e5f4a3b21* | Remove-AppxPackage"
```

## Caveats

- **Caption buttons remain visible.** The window uses `ExtendViewIntoTitleBar` + `SetTitleBar(rootGrid)` to make the entire client area draggable. 
- **First build is slow** because `windows-bindgen` generates `src/bindings.rs` from the bundled WinMD metadata. Subsequent builds are incremental.
